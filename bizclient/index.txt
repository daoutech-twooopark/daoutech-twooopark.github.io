<li>BIZCLIENT 모듈이 큐를 폴링하여 비즈뿌리오 서버로 메시지 전송</li>
<li>비즈뿌리오 서버가 발송 결과를 BIZCLIENT 모듈로 회신</li>
<li>BIZCLIENT 모듈이 <code>biz_msg</code> 의 <code>CALL_STATUS</code> 를 기록하고 <code>STATUS = 2</code> 로 변경 (비즈뿌리오 서버가 <code>biz_msg</code> 를 직접 갱신하지 않음)</li>
<li>처리 완료된 레코드는 월별 로그 테이블 <strong><code>BIZ_LOG_YYYYMM</code></strong> 으로 이동</li>
</ol>

<h3 id="rcs-발송-활성화-선택" data-spy-id="rcs-발송-활성화-선택" data-spy-section="사전-준비">RCS 발송 활성화 (선택)</h3>
<p>기본값은 RCS 미지원. RCS 를 사용하려면 <code>uds.conf</code> 에 다음을 추가:</p>
<pre><code class="language-text">MESSAGE_SUPPORT_TYPE = ALL
</code></pre>
<p>설정 후 BIZCLIENT 를 재시작하면 <code>BIZ_RCS</code> 테이블이 자동 생성됨.</p>
<h2 id="연동" data-spy-id="연동" data-spy-section="사전-준비">연동</h2>
<p><strong>BIZCLIENT 고객시스템 연동 구성</strong> — <code>biz_msg</code> / <code>BIZ_RCS</code> / <code>BIZ_ATTACHMENTS</code> 테이블의 전체 컬럼 정의</p>
<p>BIZCLIENT 는 고객사 데이터베이스에 큐 테이블을 생성하고, 고객사 애플리케이션이 발송 데이터를 INSERT 하면 모듈이 이를 감지하여 비즈뿌리오 서버로 전송하는 방식으로 동작합니다.</p>

<h3 id="1-bizclient-설치" data-spy-id="1-bizclient-설치" data-spy-section="quickstart-bizclient">1. BIZCLIENT 설치</h3>
<h3 id="linuxunix" data-spy-id="linuxunix" data-spy-section="quickstart-bizclient">Linux/Unix</h3>
<pre><code class="language-bash"># 모듈 압축 해제
unzip biz_client_v5xxx.zip
cd biz_client_v5xxx

# 환경 설정
vi config/uds.conf
</code></pre>
<p><code>uds.conf</code>:</p>
<pre><code class="language-text">UDS_ID    = bizUserId001               # bizppurio.com 계정
UDS_PW    = mypassword
DBNAME    = MYSQL                    # MYSQL / MSSQL2005 / ORACLE 등
DBURL     = jdbc:mysql://localhost:3306/bizclient
DBUSER    = bizclient
DBPASS    = mydbpassword
</code></pre>
<pre><code class="language-bash"># 시작
cd script
./biz_start
</code></pre>
<h3 id="windows" data-spy-id="windows" data-spy-section="quickstart-bizclient">Windows</h3>

notepad config\uds.conf       # 같은 항목 설정
cd bat
service-install.bat
service-start.bat
</code></pre>
<blockquote>
<p><strong>NOTE:</strong> 모듈을 처음 시작하면 <strong><code>BIZ_MSG</code></strong>, <strong><code>BIZ_LOG_YYYYMM</code></strong> 테이블이 자동 생성됩니다. 테이블 생성 권한이 있어야 합니다.</p>
</blockquote>
<h3 id="2-sms-1건-발송" data-spy-id="2-sms-1건-발송" data-spy-section="quickstart-bizclient">2. SMS 1건 발송</h3>
<p><code>BIZ_MSG</code> 테이블에 INSERT 1번이면 끝입니다.</p>
<pre><code class="language-sql">INSERT INTO biz_msg (
  MSG_TYPE, CMID, REQUEST_TIME, SEND_TIME,
  DEST_PHONE, SEND_PHONE, MSG_BODY
) VALUES (
  0,                                   -- SMS = 0
  'QS-001',                            -- 고유 키 (ASCII)
  NOW(), NOW(),                        -- 등록·발송 시간
  '01012345678',                       -- 수신번호 (본인 번호)
  '07000000000',                       -- 등록된 발신번호
  '비즈뿌리오 BIZCLIENT Quickstart 테스트입니다.'
);
</code></pre>
<p>INSERT 직후 <code>BIZ_MSG.STATUS = 0</code> (발송 대기). BIZCLIENT 모듈이 큐를 폴링하여 수 초 내에 비즈뿌리오 서버로 전송합니다.</p>
<blockquote>
<p><strong>WARNING:</strong> <strong><code>CMID</code>는 고유해야</strong> 합니다. 중복되면 발송은 되지만 결과 리포트 매핑이 누락될 수 있습니다.</p>
</blockquote>

<ul>
<li><strong>강조표기</strong> (<code>extra.title</code>): 템플릿 내용 중 강조 표기할 핵심 정보. 강조 표기 타이틀은 <strong>최대 50자</strong>, 알림톡 본문(1000자)에 <strong>포함되지 않음</strong>.</li>
<li><strong>이미지·아이템리스트</strong>: 기본 텍스트 알림톡에 (이미지 / 헤더 / 아이템 하이라이트 / 아이템리스트 / 아이템 요약정보) 5개 항목이 추가 구성. 이미지·헤더·아이템 하이라이트 영역을 필요에 따라 <strong>1개 이상 필수 선택</strong>하여 템플릿 등록. 템플릿당 고정 이미지만 사용 가능. <strong>아이템리스트는 최소 2개 ~ 최대 10개</strong>.</li>
<li><strong><code>msg_type</code></strong>: 알림톡 템플릿 강조 유형이 이미지형(IMAGE)인 경우에만 <code>msg_type</code> 을 <code>AI</code> 로 설정해야 발송 가능. (아이템리스트형이고 이미지가 포함된 경우 <code>AI</code> 로 설정 시 발송 실패)</li>
</ul>

<h2 id="대체발송" data-spy-id="대체발송" data-spy-section="가이드">대체발송</h2>
<p><strong>BIZCLIENT 대체 발송 쿼리</strong> — AT/BT + 1·2차 대체 발송 / RCS + 1·2차 대체 발송 INSERT 예제</p>
<p>발송 실패 시 다른 채널로 자동 전환. <code>RE_TYPE</code> 코드 정의는 <a href="/bizclient#용어">발송 용어 — 대체발송타입</a> 참고.</p>
<blockquote>
<p>⚠️ 대체 발송을 사용하려면 비즈뿌리오 ID 가 <strong>대체 발송 사용 가능</strong> 으로 설정되어 있어야 합니다.</p>
</blockquote>
<h3 id="매핑-키-동작" data-spy-id="매핑-키-동작" data-spy-section="가이드">매핑 키 동작</h3>
<p>대체 발송이 이루어진 경우 BIZCLIENT 는 결과 리포트 업데이트 시점에 대체 발송 정보를 <strong>추가 레코드</strong> 로 신규 생성:</p>
<ul>
<li>원본 메시지의 <code>UMID</code> ↔ 1차 대체 발송의 <code>CMID</code></li>
<li>2차 대체가 있는 경우: <code>UMID</code> 앞에 <code>RE_</code> 가 추가된 문자열이 2차 대체의 <code>CMID</code> (1차 대체의 <code>UMID</code> 도 같은 값)</li>
</ul>
<h3 id="sms-mms-대체-—-본문-동작" data-spy-id="sms-mms-대체-—-본문-동작" data-spy-section="가이드">SMS / MMS 대체 — 본문 동작</h3>
<ul>
<li><code>RE_BODY</code> 를 비워두면 원본 <code>MSG_BODY</code> 가 그대로 사용됨</li>

</ul>

<h3 id="rcs-1차-대체-발송" data-spy-id="rcs-1차-대체-발송" data-spy-section="가이드">RCS + 1차 대체 발송</h3>
<p>RCS 발송 실패 시 SMS / MMS / AT / BT 중 하나로 대체.</p>
<p><strong>RCS + SMS 대체</strong></p>
<pre><code class="language-sql">INSERT INTO biz_msg (

) VALUES (
  8, '201XXXXXXXXX', NOW(), NOW(),
  '01012345678', '0212341234',
  '{RCS_REFKEY}', 'SMS', '[다우기술] 응모하신 프로모션에 당첨되셨습니다.'
);
</code></pre>
<p><strong>RCS + MMS 대체</strong></p>
<pre><code class="language-sql">INSERT INTO biz_msg (

) VALUES (
  8, '201XXXXXXXXX', NOW(), NOW(),

</code></pre>
<p><strong>RCS + AT 대체</strong> — 알림톡 대체 발송은 <code>MSG_BODY</code> 필드의 데이터를 사용.</p>

  'K', '{RCS_REFKEY}'

<p><strong>RCS + BT 대체</strong> — 브랜드 메시지 대체도 <code>MSG_BODY</code>·<code>TEMPLATE_CODE</code> 등 채널 필수 컬럼을 그대로 포함하며 <code>RE_TYPE = U</code>.</p>

<h3 id="rcs-2차-대체-발송" data-spy-id="rcs-2차-대체-발송" data-spy-section="가이드">RCS + 2차 대체 발송</h3>
<p>RCS 발송 실패 시 1차로 AT/BT, 그것도 실패하면 2차로 SMS/MMS 발송. <code>RE_TYPE</code> 은 1차·2차 코드를 조합합니다 (<code>KS</code> = AT→SMS, <code>KM</code> = AT→MMS, <code>US</code> = BT→SMS, <code>UM</code> = BT→MMS).</p>
<blockquote>
<p>1차가 AT/BT 이므로 <strong><code>MSG_BODY</code> 에 1차(AT/BT) 본문</strong>, <strong><code>RE_BODY</code> 에 2차(SMS/MMS) 본문</strong>을 넣습니다. <code>TEMPLATE_CODE</code>·<code>SENDER_KEY</code>·<code>NATION_CODE</code> 등 1차 채널의 필수 컬럼을 모두 포함해야 합니다.</p>
</blockquote>
